🐾 Pet Shelter Manager

Build a complete Python class system to manage a pet adoption shelter

Python 101 • Learn and Help Program • Metropolitan State University

📋 Scenario

You've been hired to build a management system for Happy Tails Pet Shelter. The shelter takes in animals, tracks their info, and manages adoptions. Your job is to write a Pet class that models each animal and keeps track of shelter-wide statistics.

🎯

This assignment requires you to use all the OOP concepts from the playbook: classes, objects, instance variables, class variables, instance methods, class methods, static methods, and magic methods (__init__, __str__, __repr__).

1 Define the Pet Class ⭐ 15 pts

Create a class called Pet with the following class variables:

  • total_pets — starts at 0, tracks how many Pet objects have been created
  • adopted_count — starts at 0, tracks how many pets have been adopted
  • shelter_name — set to "Happy Tails Shelter"

What you're practicing

Class variables, the class keyword, and understanding shared vs. per-object data.

2 Write the __init__ Constructor ⭐ 15 pts

The constructor should accept and store these instance variables:

  • name — the pet's name (e.g., "Buddy")
  • species — "Dog", "Cat", "Rabbit", etc.
  • age — age in years (can be a float, e.g. 0.5 for 6 months)
  • is_adopted — defaults to False

The constructor should also increment total_pets by 1 every time a new Pet is created.

💡

Remember: self.name is an instance variable (unique per pet). Pet.total_pets is a class variable (shared by all pets). Update the class variable using Pet.total_pets, not self.total_pets.

3 Add __str__ and __repr__ ⭐ 15 pts

__str__ — User-friendly display

Should return a string like:

Example __str__ output Buddy (Dog) - Age: 3 years - Status: Available

If the pet has been adopted, show "Status: Adopted 🏠" instead of "Status: Available".

__repr__ — Developer-friendly display

Should return a string that could recreate the object:

Example __repr__ output Pet('Buddy', 'Dog', 3, False)

4 Instance Methods ⭐ 20 pts

Add these three instance methods to your Pet class:

a) adopt(owner_name) 8 pts

  • If the pet is already adopted, print: "Buddy has already been adopted!"
  • Otherwise, set is_adopted to True, store the owner_name as a new instance variable self.owner, increment the class variable adopted_count, and print: "🎉 Buddy has been adopted by Sarah!"

b) get_info() 6 pts

  • Returns a dictionary with the pet's data:
    {"name": "Buddy", "species": "Dog", "age": 3, "adopted": False}
  • If the pet has been adopted, also include "owner": "Sarah"

c) celebrate_birthday() 6 pts

  • Increases age by 1
  • Prints: "🎂 Happy Birthday, Buddy! Now 4 years old."

5 Class Method & Static Method ⭐ 15 pts

a) @classmethod shelter_stats(cls) 10 pts

Returns a formatted string with shelter-wide statistics:

Example output 📊 Happy Tails Shelter Stats: Total pets: 5 | Adopted: 2 | Available: 3

b) @staticmethod is_senior(species, age) 5 pts

A utility method that returns True if the pet is a senior based on these rules:

  • Dogs: age ≥ 7
  • Cats: age ≥ 10
  • All others: age ≥ 5

6 Test Your Class ⭐ 20 pts

Write a test script (in the same file, below the class) that demonstrates ALL features. Your script must:

  1. Create at least 3 Pet objects with different species 4 pts
  2. Print each pet using print() to show __str__ works 2 pts
  3. Print repr() of one pet to show __repr__ works 2 pts
  4. Adopt at least 2 pets — then try adopting one again to show the error message 3 pts
  5. Call celebrate_birthday() on one pet 2 pts
  6. Call get_info() on an adopted and a non-adopted pet, and print both 3 pts
  7. Print the shelter_stats() class method 2 pts
  8. Use is_senior() to check at least 2 pets and print the results 2 pts

🔑 Helpful Reminders

1️⃣

self refers to the specific object. Use self.name to access that pet's name.

2️⃣

cls refers to the class itself. Use cls.total_pets inside a @classmethod.

3️⃣

To update a class variable, use Pet.total_pets += 1 — if you use self.total_pets += 1, it creates an instance variable that shadows the class variable!

4️⃣

__str__ is for print(). __repr__ is for developers. If only __repr__ exists, print() uses it as a fallback.

5️⃣

A @staticmethod receives neither self nor cls. It's just a regular function that lives inside the class.
