What Is a Class?

A class is a blueprint (or template) for creating objects. Think of it like the design plans for a car — the plans describe what features every car will have (color, fuel level, model), but the plans themselves are not a car.

💡

A class defines attributes (data) and methods (actions). An object is a specific instance built from that blueprint.

class_basics.py 
# Defining a simple class
class Car:
    # The __init__ method runs when an object is created
    def __init__(self, make, model, color):
        self.make  = make     # instance attribute
        self.model = model    # instance attribute
        self.color = color    # instance attribute
        self.fuel  = 0        # starts empty

What Is an Object (Instance)?

An object is a concrete thing created from the class blueprint. Each object has its own copy of the data.

creating_objects.py 
# Creating objects (instances) from the Car class
my_car   = Car("Toyota", "Camry", "Blue")
your_car = Car("Honda", "Civic", "Red")

print(my_car.make)    # Toyota
print(your_car.color) # Red

# Each object is independent
print(my_car is your_car)  # False
Output Toyota Red False

The self Parameter

Every method inside a class receives self as its first parameter. It refers to the specific object calling the method — it's how each car "knows" its own data.

🔑

self is not a keyword — it's a strong convention. Python automatically passes the calling object as the first argument.

self_demo.py 
class Car:
    def __init__(self, make, model):
        self.make  = make
        self.model = model

    def describe(self):
        # self.make refers to THIS car's make
        return f"{self.make} {self.model}"

car_a = Car("Ford", "Mustang")
car_b = Car("Tesla", "Model 3")

print(car_a.describe())  # Ford Mustang
print(car_b.describe())  # Tesla Model 3

What Are Magic Methods?

Magic methods (also called dunder methods — short for "double underscore") are special methods Python calls automatically in certain situations. They let you customize how objects behave with built-in operations.

✨

They're "magic" because you never call them directly — Python calls them for you when you use print(), +, len(), etc.

__init__(self, ...) — The Constructor

Called automatically when you create a new object. It initializes the object's attributes.

init_method.py 
class Car:
    def __init__(self, make, model, year):
        self.make  = make
        self.model = model
        self.year  = year
        self.fuel  = 0
        print(f"A new {make} {model} just rolled off the lot!")

# __init__ runs automatically here:
car = Car("Toyota", "Camry", 2024)
Output A new Toyota Camry just rolled off the lot!

__str__(self) — The Human-Friendly String

Called when you use print() or str() on an object. It returns a user-friendly description.

str_method.py 
class Car:
    def __init__(self, make, model, year):
        self.make  = make
        self.model = model
        self.year  = year

    def __str__(self):
        return f"{self.year} {self.make} {self.model}"

car = Car("Honda", "Civic", 2023)
print(car)         # calls __str__ automatically
print(str(car))    # also calls __str__
Output 2023 Honda Civic 2023 Honda Civic

__repr__(self) — The Developer String

Called when you type the variable name in the Python shell, or use repr(). It returns a developer-friendly representation — ideally one that could recreate the object.

repr_method.py 
class Car:
    def __init__(self, make, model, year):
        self.make  = make
        self.model = model
        self.year  = year

    def __str__(self):
        return f"{self.year} {self.make} {self.model}"

    def __repr__(self):
        return f"Car('{self.make}', '{self.model}', {self.year})"

car = Car("Ford", "Mustang", 2024)

print(str(car))    # user-friendly → __str__
print(repr(car))   # developer-friendly → __repr__
Output 2024 Ford Mustang Car('Ford', 'Mustang', 2024)

__str__ vs __repr__ — Quick Comparison

Feature__str____repr__
AudienceEnd usersDevelopers
Called byprint(), str()Python shell, repr()

🎯

Rule of thumb: Always define __repr__. Define __str__ only when you want a friendlier output. If only __repr__ exists, print() will use it as a fallback.

Instance Variables

Instance variables belong to a specific object. Each object gets its own copy, and changing one object's variable does NOT affect others.

self.variable UNIQUE per object
instance_vars.py 
class Car:
    def __init__(self, make, color):
        # Instance variables — unique to each car
        self.make  = make
        self.color = color
        self.fuel  = 0

car1 = Car("Toyota", "Blue")
car2 = Car("Honda", "Red")

# Each car has its own fuel level
car1.fuel = 50
print(car1.fuel)  # 50
print(car2.fuel)  # 0  — not affected!
Output 50 0

Class Variables

Class variables are shared by ALL objects of that class. They're defined directly inside the class body (not inside any method). Useful for data that's the same for every instance.

Defined at class level SHARED by all objects
class_vars.py 
class Car:
    # Class variable — shared by all Car objects
    total_cars = 0
    wheels     = 4

    def __init__(self, make, model):
        self.make  = make       # instance variable
        self.model = model      # instance variable
        Car.total_cars += 1     # update the class variable

car1 = Car("Toyota", "Camry")
car2 = Car("Honda", "Civic")
car3 = Car("Ford", "F-150")

print(Car.total_cars)    # 3
print(car1.wheels)       # 4
print(car2.wheels)       # 4  — same for all
Output 3 4 4

⚠️ Common Pitfall

If you assign to an instance with the same name as a class variable, you create a new instance variable that shadows the class variable — the class variable itself stays unchanged.

pitfall.py 
class Car:
    wheels = 4

car1 = Car()
car2 = Car()

# This creates an INSTANCE variable on car1 only
car1.wheels = 6

print(car1.wheels)     # 6  — instance variable
print(car2.wheels)     # 4  — still the class variable
print(Car.wheels)      # 4  — class variable unchanged
⚠️

To change a class variable for all objects, always use ClassName.variable = value, not self.variable = value.

Instance Methods

Instance methods are the most common type. They take self as the first parameter, which gives them access to instance (and class) variables.

def method(self) Can access self.* and ClassName.*
instance_methods.py 
class Car:
    def __init__(self, make, model):
        self.make  = make
        self.model = model
        self.fuel  = 0

    # Instance method: fills up the gas tank
    def fillGas(self, gallons):
        self.fuel += gallons
        print(f"Added {gallons} gal. Fuel: {self.fuel} gal")

    # Instance method: checks if drivable
    def canDrive(self):
        return self.fuel > 0

    # Instance method: drives and uses fuel
    def drive(self, miles):
        if self.canDrive():
            fuel_used = miles / 30  # 30 mpg
            self.fuel -= fuel_used
            print(f"Drove {miles} mi. Fuel left: {self.fuel:.1f} gal")
        else:
            print("Can't drive — no fuel!")

car = Car("Toyota", "Camry")
car.drive(10)       # Can't drive — no fuel!
car.fillGas(10)     # Added 10 gal. Fuel: 10 gal
car.drive(90)       # Drove 90 mi. Fuel left: 7.0 gal
Output Can't drive — no fuel! Added 10 gal. Fuel: 10 gal Drove 90 mi. Fuel left: 7.0 gal

Class Methods

Class methods use the @classmethod decorator and receive cls (the class itself) instead of self. They can access and modify class variables but NOT instance variables.

@classmethod def method(cls)
class_methods.py 
class Car:
    total_cars = 0

    def __init__(self, make, model):
        self.make  = make
        self.model = model
        Car.total_cars += 1

    @classmethod
    def get_total_cars(cls):
        return cls.total_cars

    @classmethod
    def from_string(cls, car_string):
        # Alternative constructor: "Toyota-Camry"
        make, model = car_string.split("-")
        return cls(make, model)

car1 = Car("Toyota", "Camry")
car2 = Car.from_string("Honda-Civic")  # using class method

print(car2.make)                    # Honda
print(Car.get_total_cars())          # 2
Output Honda 2

Static Methods (Bonus)

Static methods use @staticmethod and receive neither self nor cls. They're utility functions that live inside the class for organizational purposes.

static_methods.py 
class Car:
    @staticmethod
    def mph_to_kph(mph):
        return mph * 1.60934

print(Car.mph_to_kph(60))  # 96.5604

Complete Car Class — Putting It All Together

Here's a complete Car class that uses everything we've learned: instance variables, class variables, instance methods, class methods, and magic methods.

car_complete.py 
class Car:
    # ── Class Variables ──
    total_cars = 0
    wheels     = 4

    # ── Constructor (Magic Method) ──
    def __init__(self, make, model, year, color):
        # Instance variables
        self.make  = make
        self.model = model
        self.year  = year
        self.color = color
        self.fuel  = 0         # gallons
        self.odometer = 0     # miles driven
        Car.total_cars += 1

    # ── Magic Methods ──
    def __str__(self):
        return f"{self.year} {self.make} {self.model} ({self.color})"

    def __repr__(self):
        return (f"Car('{self.make}', '{self.model}', "
                f"{self.year}, '{self.color}')")

    # ── Instance Methods ──
    def fillGas(self, gallons):
        self.fuel += gallons
        print(f"⛽ Filled {gallons} gal → Tank: {self.fuel} gal")

    def canDrive(self):
        return self.fuel > 0

    def drive(self, miles):
        if not self.canDrive():
            print("🚫 Can't drive — tank is empty!")
            return
        fuel_needed = miles / 30
        if fuel_needed > self.fuel:
            actual_miles = self.fuel * 30
            self.odometer += actual_miles
            self.fuel = 0
            print(f"🚗 Drove {actual_miles:.0f} mi and ran out of gas!")
        else:
            self.fuel -= fuel_needed
            self.odometer += miles
            print(f"🚗 Drove {miles} mi → Fuel: {self.fuel:.1f} gal")

    def status(self):
        print(f"── {self} ──")
        print(f"   Fuel: {self.fuel} gal | Odometer: {self.odometer} mi")

    # ── Class Method ──
    @classmethod
    def get_total_cars(cls):
        return f"Total cars on the road: {cls.total_cars}"

    # ── Static Method ──
    @staticmethod
    def mph_to_kph(mph):
        return round(mph * 1.60934, 2)


# ═══════════════════════════════════════════
# Let's take our cars for a spin!
# ═══════════════════════════════════════════

car1 = Car("Toyota", "Camry", 2024, "Silver")
car2 = Car("Tesla", "Model 3", 2025, "White")

print(car1)              # __str__
print(repr(car2))        # __repr__

car1.drive(50)           # Can't drive — empty!
car1.fillGas(15)         # Fill up
car1.drive(120)          # Go for a drive
car1.drive(400)          # Run out of gas
car1.status()

print(Car.get_total_cars())
print(f"60 mph = {Car.mph_to_kph(60)} kph")
Output 2024 Toyota Camry (Silver) Car('Tesla', 'Model 3', 2025, 'White') 🚫 Can't drive — tank is empty! ⛽ Filled 15 gal → Tank: 15 gal 🚗 Drove 120 mi → Fuel: 11.0 gal 🚗 Drove 330 mi and ran out of gas! ── 2024 Toyota Camry (Silver) ── Fuel: 0 gal | Odometer: 450 mi Total cars on the road: 2 60 mph = 96.56 kph
