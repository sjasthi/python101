üìö Introduction to Function Arguments

What are Function Arguments?

Function arguments are values passed to a function when it's called. They provide data that the function needs to perform its task. Python offers multiple ways to pass arguments, each with its own advantages!

Key Terms:
  • Parameter: Variable in the function definition (blueprint)
  • Argument: Actual value passed when calling the function (data)
# Example of parameters vs arguments def greet(name, age): # 'name' and 'age' are parameters print(f"Hello {name}, you are {age} years old!") greet("Alice", 15) # "Alice" and 15 are arguments

üéØ Positional Arguments

Understanding Positional Arguments

Positional arguments are matched to parameters based on their position (order) in the function call. The first argument goes to the first parameter, the second to the second, and so on.

  • Order matters! Changing the order changes the meaning.
  • Most common and straightforward way to pass arguments.
  • Number of arguments must match the number of parameters.
# Positional arguments example def calculate_rectangle_area(length, width): return length * width # Correct order: length=5, width=3 area1 = calculate_rectangle_area(5, 3) print(area1) # Output: 15 # Different order gives different result! area2 = calculate_rectangle_area(3, 5) print(area2) # Output: 15 (same here, but concept matters!)

üîë Keyword Arguments

Understanding Keyword Arguments

Keyword arguments are passed with parameter names explicitly specified. This makes the code more readable and allows you to pass arguments in any order!

  • Use the syntax: parameter_name=value
  • Order doesn't matter when using keyword arguments
  • Makes code more readable and self-documenting
  • Helps prevent errors from wrong argument order
# Keyword arguments example def book_flight(passenger, destination, date, seat_class): return f"{passenger} booked {seat_class} to {destination} on {date}" # Using keyword arguments - order doesn't matter! booking1 = book_flight( passenger="John", destination="Paris", date="2025-12-15", seat_class="Business" ) # Same result with different order booking2 = book_flight( date="2025-12-15", seat_class="Business", destination="Paris", passenger="John" )

üîÑ Mixing Positional and Keyword Arguments

You can use both positional and keyword arguments in the same function call, but positional arguments MUST come before keyword arguments!

Rule: Positional arguments first, then keyword arguments.
# Mixing positional and keyword arguments def create_profile(username, email, age, city): print(f"User: {username}, Email: {email}, Age: {age}, City: {city}") # ‚úÖ Correct: Positional first, then keyword create_profile("alex123", "alex@email.com", age=17, city="Boston") # ‚ùå Wrong: Keyword before positional causes error! # create_profile(username="alex123", "alex@email.com", 17, "Boston")

‚≠ê Optional Arguments (Default Parameters)

Understanding Optional Arguments

Optional arguments have default values assigned in the function definition. If you don't provide a value when calling the function, the default value is used automatically!

  • Define with syntax: parameter=default_value
  • Make functions more flexible and easier to use
  • Optional parameters must come after required parameters
  • Can be overridden by passing a new value
# Optional arguments example def send_message(recipient, message, priority="normal", urgent=False): print(f"To: {recipient}") print(f"Message: {message}") print(f"Priority: {priority}, Urgent: {urgent}") # Call with only required arguments - defaults are used send_message("Alice", "Hello!") # Override one default send_message("Bob", "Meeting at 3pm", priority="high") # Override both defaults send_message("Carol", "Emergency!", priority="critical", urgent=True)

üìã Default Values Best Practices

  • Use immutable defaults: Prefer numbers, strings, None, True/False
  • Document defaults: Make it clear what default behavior is
  • Sensible defaults: Choose defaults that work for most cases
  • Avoid mutable defaults: Don't use lists or dictionaries as defaults (advanced topic!)
