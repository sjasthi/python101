Chapter 9 · Dictionaries & Sets

Python Sets { }

Unordered collections of unique items — your toolkit for removing duplicates and comparing groups!

📊 Lists vs Tuples vs Sets

You already know lists and tuples. Here's how sets compare:

FeatureLists [ ]Tuples ( )Sets { }
Ordered✔✔✘
Indexed✔✔✘
Add / Update items✔✘✔
Can contain duplicates✔✔✘
UsesSquare BracketsRound BracketsCurly Braces

Key takeaway: Sets are unordered, unindexed, and do not allow duplicates. Think of a set like a bag of unique items — no order, no repeats!

🛠️ Creating a Set

Using Curly Braces

Python
fruits = {"apple", "banana", "grapes", "mango"}
print(fruits)
print(type(fruits))
Output
{'mango', 'apple', 'banana', 'grapes'}
<class 'set'>

Creating an Empty Set — Watch Out!

Python
# Common mistake!
wrong = {}
print(type(wrong))    # <class 'dict'>  — NOT a set!

# Correct way:
correct = set()
print(type(correct))  # <class 'set'>
Output
<class 'dict'>
<class 'set'>
⚠️ Watch out: {} creates an empty dictionary, NOT an empty set. Always use set() for an empty set!

🚫 Sets Automatically Remove Duplicates

If you add duplicate values, Python keeps only one copy.

Python
set_x = {10, 20, 20, 20, 20, 30}
print(set_x)
print(*set_x)
Output
{10, 20, 30}
10 20 30

A marks list with duplicates cannot be properly represented as a set:

Python
# marks = [10, 8, 7, 0, 9, 11, 10, 10, 4, 5]
# Converting to a set loses the duplicate 10s!
marks_set = {10, 8, 7, 0, 9, 11, 10, 10, 4, 5}
print(marks_set)
Output
{0, 4, 5, 7, 8, 9, 10, 11}
💡 Pro tip: Converting a list to a set is a quick way to find unique values!

🔀 Sets Are Unordered & Unindexed

Unlike lists and tuples, sets have no order and no index positions. You cannot access elements by index.

Python
set_y = {"amy", "barb", "chris", "dave"}
first_student = set_y[0]   # This will CRASH!
Output
TypeError: 'set' object is not subscriptable
⚠️ Remember: Since sets are unordered, there is no "first" or "last" item. Every time you print a set, the order may be different!

🔒 Sets Only Support Immutable Data Types

Set items must be immutable (unchangeable). You can store numbers, strings, booleans, and tuples — but not lists.

Why Can't You Put a List in a Set?

Python
A = [1, 2, 3]
B = [2, 3, 4]
set_x = {A, B}   # This will CRASH!
Output
TypeError: unhashable type: 'list'

Think about it: If lists were allowed, you could modify B to become [1,2,3] — creating duplicates inside the set. That violates the set rule!

But Tuples Work Just Fine!

Python
A = (1, 2, 3)
B = (2, 3, 4)
set_x = {A, B}
print(set_x)
Output
{(1, 2, 3), (2, 3, 4)}
💡 Key rule: Only immutable items (numbers, strings, booleans, tuples) can go inside a set — not lists.

🎨 A Single Set Can Contain Multiple Data Types

Python
# student name, age, email, is_active?
student_info = {"John Doe", 15, "john.doe@gmail.com", True}
print(student_info)
Output
{'John Doe', 'john.doe@gmail.com', True, 15}
Notice: The order in the output is different from how we typed it! Order is not guaranteed in sets.

📝 CRUD Operations on Sets

Remember the CRUD framework? Here's how sets compare:

Create — Building a Set

Python
# Method 1: Create with initial values
students = {"abe", "barb", "chris"}
print(students)

# Method 2: Start empty, then add
students = set()
students.add("abe")
students.add("barb")
students.add("chris")
students.add("dave")
print(*students)
Output
{'barb', 'abe', 'chris'}
dave barb abe chris

➕ Adding & ➖ Removing Items

Adding Items

Python
# add() — add one element
students = {"abe", "barb", "chris"}
students.add("fiona")
print("After add:", students)

# update() — add multiple elements (from list, tuple, or set)
more = ["greg", "hannah"]
students.update(more)
print("After update:", students)
Output
After add: {'fiona', 'barb', 'abe', 'chris'}
After update: {'fiona', 'barb', 'greg', 'abe', 'hannah', 'chris'}

Removing Items — Three Ways

Python
fruits = {"apple", "banana", "cherry", "mango"}

# remove(x) — raises KeyError if not found
fruits.remove("banana")
print("After remove:", fruits)

# discard(x) — NO error if not found
fruits.discard("grape")     # no error!
print("After discard:", fruits)

# pop() — removes a RANDOM element
removed = fruits.pop()
print("Popped:", removed)
print("Remaining:", fruits)
Output
After remove: {'apple', 'cherry', 'mango'}
After discard: {'apple', 'cherry', 'mango'}
Popped: apple
Remaining: {'cherry', 'mango'}
💡 Tip: Use discard() when you're not sure if the item exists. Use remove() when you expect it to be there.

🦸 Set Operations — The Superpower of Sets!

This is where sets truly shine. Let's use the Mammals vs Aquatic animals scenario:

Python — Setup
# Two sets — one for mammals, one for aquatic animals
mammals = {"Tiger", "Camel", "Sheep", "Whale", "Walrus"}
aquatic = {"Octopus", "Squid", "Crab", "Whale", "Walrus"}

Union — All animals from EITHER group ( | or .union() )

The union of A and B is the set of elements in A, B, or both (A ∪ B).

Python
mammals = {"Tiger", "Camel", "Sheep", "Whale", "Walrus"}
aquatic = {"Octopus", "Squid", "Crab", "Whale", "Walrus"}

# union method
animals = mammals.union(aquatic)
print("Union (method):", animals)

# | operator
animals = mammals | aquatic
print("Union (|):     ", animals)

# Original sets are NOT modified
print("mammals:", mammals)
print("aquatic:", aquatic)
Output
Union (method): {'Octopus', 'Tiger', 'Sheep', 'Walrus', 'Whale', 'Crab', 'Camel', 'Squid'}
Union (|):      {'Octopus', 'Tiger', 'Sheep', 'Walrus', 'Whale', 'Crab', 'Camel', 'Squid'}
mammals: {'Tiger', 'Sheep', 'Walrus', 'Whale', 'Camel'}
aquatic: {'Octopus', 'Walrus', 'Crab', 'Whale', 'Squid'}

Intersection — Animals in BOTH groups ( & or .intersection() )

The intersection of A and B is the set of elements common to both (A ∩ B).

Python
mammals = {"Tiger", "Camel", "Sheep", "Whale", "Walrus"}
aquatic = {"Octopus", "Squid", "Crab", "Whale", "Walrus"}

animals = mammals.intersection(aquatic)
print("Intersection:", animals)

# Using & operator
animals = mammals & aquatic
print("Using &:     ", animals)
Output
Intersection: {'Walrus', 'Whale'}
Using &:      {'Walrus', 'Whale'}

Difference — In one group but NOT the other ( - or .difference() )

The difference of A and B is the set of elements in A that are not in B (A - B). Note: A-B ≠ B-A!

Python
mammals = {"Tiger", "Camel", "Sheep", "Whale", "Walrus"}
aquatic = {"Octopus", "Squid", "Crab", "Whale", "Walrus"}

# mammals - aquatic (land-only mammals)
mammals_only = mammals.difference(aquatic)
print("mammals - aquatic:", mammals_only)

# aquatic - mammals (aquatic-only animals)
aquatic_only = aquatic.difference(mammals)
print("aquatic - mammals:", aquatic_only)
Output
mammals - aquatic: {'Sheep', 'Tiger', 'Camel'}
aquatic - mammals: {'Octopus', 'Squid', 'Crab'}

Symmetric Difference — In one group but NOT both ( ^ or .symmetric_difference() )

The symmetric difference is the set of elements in either A or B, but not in both (A △ B).

Python
mammals = {"Tiger", "Camel", "Sheep", "Whale", "Walrus"}
aquatic = {"Octopus", "Squid", "Crab", "Whale", "Walrus"}

exclusive = mammals.symmetric_difference(aquatic)
print("Symmetric diff:", exclusive)

# Using ^ operator
exclusive = mammals ^ aquatic
print("Using ^:       ", exclusive)
Output
Symmetric diff: {'Sheep', 'Octopus', 'Crab', 'Camel', 'Tiger', 'Squid'}
Using ^:        {'Sheep', 'Octopus', 'Crab', 'Camel', 'Tiger', 'Squid'}

Quick Summary

OperationMethodOperatorResult
Union.union()|ALL animals
Intersection.intersection()&BOTH groups
Difference.difference()-One group ONLY
Symmetric Diff.symmetric_difference()^NOT in both

🧰 Built-in Functions on Sets

Python
numbers = {3, 1, 4, 1, 5, 9, 2, 6}
print("Set:   ", numbers)   # duplicates removed!
print("len:   ", len(numbers))
print("min:   ", min(numbers))
print("max:   ", max(numbers))
print("sum:   ", sum(numbers))
print("sorted:", sorted(numbers))
Output
Set:    {1, 2, 3, 4, 5, 6, 9}
len:    7
min:    1
max:    9
sum:    30
sorted: [1, 2, 3, 4, 5, 6, 9]
💡 Note: sorted() returns a list, not a set — but it still works on a set as input!
