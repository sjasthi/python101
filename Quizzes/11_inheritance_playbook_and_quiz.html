What Is a Class?

A class is like a blueprint or a cookie cutter. It describes what something looks like and what it can do, but it isn't the actual thing.

An object (also called an instance) is what you create from the blueprint — it's the actual cookie.

TermAnalogyPython
ClassBlueprint / Cookie cutterclass Dog:
ObjectThe actual house / cookiemy_dog = Dog()
AttributeCharacteristics (color, size)self.name
MethodActions (run, bark, eat)def bark(self):

Defining a Class

Use the class keyword, then define __init__ to set up each object's starting attributes:

python 
class Dog:
    def __init__(self, name, breed):
        self.name = name      # instance attribute
        self.breed = breed    # instance attribute

    def bark(self):
        return f"{self.name} says Woof!"

    def __str__(self):
        return f"{self.name} the {self.breed}"
💡 self — refers to the specific object being created. Every method's first parameter must be self.

Creating Objects (Instances)

Call the class name like a function, passing the values that __init__ expects:

python 
# Create two Dog objects from the same class
dog1 = Dog("Buddy", "Golden Retriever")
dog2 = Dog("Luna", "Poodle")

print(dog1.bark())     # Buddy says Woof!
print(dog2.bark())     # Luna says Woof!
print(dog1.name)       # Buddy
print(dog2.breed)      # Poodle
print(dog1)            # Buddy the Golden Retriever
Buddy says Woof! Luna says Woof! Buddy Poodle Buddy the Golden Retriever
🔑 Key idea: One class → many objects. Each object has its own attribute values but shares the same methods.

Why Do We Need Classes?

Classes help you organize your code. Instead of writing separate variables and functions for every dog, you bundle everything together:

  • Without classes: dog1_name, dog1_breed, dog1_bark(), dog2_name… messy!
  • With classes: dog1.name, dog1.bark() — clean and organized!

Now that you understand classes, you're ready for the big idea: what if one class could borrow everything from another class? That's inheritance — head to the next tab! →

What Is Inheritance?

Inheritance lets a new class reuse attributes and methods from an existing class — like how you inherit traits from your parents but also develop your own!

TermAlso CalledRole
Parent classBase class, SuperclassThe class being inherited from
Child classDerived class, SubclassThe class that inherits
🎯 Why use inheritance? Write code once in the parent → reuse it in many children. Fix a bug in one place → all children benefit. Add new features without changing the parent.

Basic Syntax

Put the parent class name in parentheses after the child class name:

python 
class Animal:              # Parent class
    def __init__(self, name, sound):
        self.name = name
        self.sound = sound

    def speak(self):
        return f"{self.name} says {self.sound}!"


class Dog(Animal):           # Child class — inherits from Animal
    def fetch(self, item):
        return f"{self.name} fetches the {item}!"


class Cat(Animal):           # Another child class
    def purr(self):
        return f"{self.name} is purring..."
using it 
dog = Dog("Buddy", "Woof")
cat = Cat("Whiskers", "Meow")

print(dog.speak())       # Buddy says Woof!       ← inherited
print(dog.fetch("ball")) # Buddy fetches the ball! ← Dog's own
print(cat.speak())       # Whiskers says Meow!     ← inherited
print(cat.purr())        # Whiskers is purring...  ← Cat's own
Buddy says Woof! Buddy fetches the ball! Whiskers says Meow! Whiskers is purring...
🔑 Notice: Dog and Cat never defined __init__ or speak(), but they inherited both from Animal. Each child also added its own unique method.

The super() Function

When a child class needs its own __init__ to add new attributes, use super() to call the parent's __init__ first:

python 
class Animal:
    def __init__(self, name, sound):
        self.name = name
        self.sound = sound

    def speak(self):
        return f"{self.name} says {self.sound}!"


class Dog(Animal):
    def __init__(self, name, sound, breed):
        super().__init__(name, sound)   # ← call parent's __init__
        self.breed = breed              # ← add new attribute

    def info(self):
        return f"{self.name} is a {self.breed}"
using it 
dog = Dog("Buddy", "Woof", "Golden Retriever")

print(dog.speak())   # Buddy says Woof!            ← inherited
print(dog.info())    # Buddy is a Golden Retriever  ← Dog's own
print(dog.breed)     # Golden Retriever             ← new attribute
Buddy says Woof! Buddy is a Golden Retriever Golden Retriever
⚠️ Common mistake: Forgetting super().__init__() inside the child's __init__. Without it, the parent's attributes (name, sound) won't be set and you'll get an AttributeError!

Method Overriding

A child class can replace a parent's method by defining a method with the same name:

python 
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} makes a sound"


class Dog(Animal):
    def speak(self):                        # ← overrides Animal.speak()
        return f"{self.name} says Woof!"


class Cat(Animal):
    def speak(self):                        # ← overrides Animal.speak()
        return f"{self.name} says Meow!"
using it 
animal = Animal("Generic")
dog    = Dog("Buddy")
cat    = Cat("Whiskers")

print(animal.speak())   # Generic makes a sound
print(dog.speak())      # Buddy says Woof!
print(cat.speak())      # Whiskers says Meow!
Generic makes a sound Buddy says Woof! Whiskers says Meow!
💡 Key idea: Same method name, different behavior in each class. Python always uses the child's version when available.

isinstance() and issubclass()

Python gives you two built-in functions to check inheritance relationships:

python 
dog = Dog("Buddy")

# isinstance — is this object an instance of this class?
print(isinstance(dog, Dog))      # True  — dog IS a Dog
print(isinstance(dog, Animal))   # True  — dog IS ALSO an Animal
print(isinstance(dog, Cat))      # False — dog is NOT a Cat

# issubclass — is this class a child of that class?
print(issubclass(Dog, Animal))   # True
print(issubclass(Cat, Animal))   # True
print(issubclass(Dog, Cat))      # False
True True False True True False

Real-World Example — School System

A practical example showing how inheritance organizes related classes:

python 
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        return f"Hi, I'm {self.name}, age {self.age}"


class Student(Person):
    def __init__(self, name, age, student_id, major):
        super().__init__(name, age)
        self.student_id = student_id
        self.major = major

    def introduce(self):
        return f"Hi, I'm {self.name}, {self.major} major (ID: {self.student_id})"


class Teacher(Person):
    def __init__(self, name, age, department):
        super().__init__(name, age)
        self.department = department

    def introduce(self):
        return f"Hi, I'm {self.name}, {self.department} department"
using it 
s = Student("Alice", 20, "S1001", "Computer Science")
t = Teacher("Dr. Smith", 45, "Math")

print(s.introduce())   # Hi, I'm Alice, Computer Science major (ID: S1001)
print(t.introduce())   # Hi, I'm Dr. Smith, Math department
print(s.age)            # 20  ← inherited from Person
Hi, I'm Alice, Computer Science major (ID: S1001) Hi, I'm Dr. Smith, Math department 20
