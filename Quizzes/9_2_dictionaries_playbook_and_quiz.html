📊 Lists vs Tuples vs Sets vs Dictionaries

You already know lists, tuples, and sets. Here's how dictionaries compare:

Key takeaway: Dictionaries store data as key : value pairs. Each key must be unique, but values can repeat. Think of a real dictionary — each word (key) maps to a definition (value)!

📖 What Is a Dictionary?

A dictionary is a collection of key : value pairs wrapped in curly braces. Each key acts as a label that points to a value.

Real-World Analogies

  • Phone contacts: Name (key) → Phone number (value)
  • Student grades: Student name (key) → Grade (value)
  • Menu prices: Food item (key) → Price (value)
  • Country capitals: Country (key) → Capital city (value)
Python
# A dictionary of student grades
student = {"name": "Alice", "age": 14, "grade": "8th"}
print(student)
print(type(student))
Output
{'name': 'Alice', 'age': 14, 'grade': '8th'}
<class 'dict'>
💡 Remember: {} creates an empty dictionary (not a set!). To create an empty set, you need set().

🛠️ Creating a Dictionary

Method 1: Using Curly Braces

Python
# Create with initial key:value pairs
capitals = {
    "USA": "Washington D.C.",
    "France": "Paris",
    "Japan": "Tokyo"
}
print(capitals)
Output
{'USA': 'Washington D.C.', 'France': 'Paris', 'Japan': 'Tokyo'}

Method 2: Using dict() Constructor

Python
# Using the dict() constructor (note: no quotes on keys!)
capitals = dict(USA="Washington D.C.", France="Paris", Japan="Tokyo")
print(capitals)
Output
{'USA': 'Washington D.C.', 'France': 'Paris', 'Japan': 'Tokyo'}

Method 3: Starting Empty

Python
# Start empty, then add pairs
menu = {}
menu["burger"] = 5.99
menu["fries"] = 2.99
menu["soda"] = 1.49
print(menu)
Output
{'burger': 5.99, 'fries': 2.99, 'soda': 1.49}

🚫 Duplicate Keys Are NOT Allowed

If you use the same key twice, the last value wins. Python silently overwrites the earlier one.

Python
# "apple" appears twice — what happens?
prices = {"apple": 1.50, "banana": 0.75, "apple": 2.00}
print(prices)
print(len(prices))
Output
{'apple': 2.00, 'banana': 0.75}
2
⚠️ Watch out: Duplicate keys don't cause an error — they just overwrite! The dictionary ends up with only 2 items, not 3.

🔍 Reading / Accessing Values

Access values using their key — not a numeric index!

Method 1: Square Brackets

Python
student = {"name": "Alice", "age": 14, "grade": "8th"}

print(student["name"])
print(student["age"])
Output
Alice
14

What If the Key Doesn't Exist?

Python
student = {"name": "Alice", "age": 14, "grade": "8th"}
print(student["email"])   # KeyError!
Output
KeyError: 'email'

Method 2: .get() — The Safe Way

Python
student = {"name": "Alice", "age": 14, "grade": "8th"}

# .get() returns None if key doesn't exist (no crash!)
print(student.get("name"))
print(student.get("email"))

# You can set a default value
print(student.get("email", "Not provided"))
Output
Alice
None
Not provided
💡 Tip: Use .get() when you're not sure if the key exists. Use [] when you're certain it does.

✏️ Updating & Adding Items

Adding and updating use the same syntax. If the key exists, it updates. If it doesn't, it adds a new pair.

Add / Update with Square Brackets

Python
student = {"name": "Alice", "age": 14}

# Update an existing key
student["age"] = 15
print("After update:", student)

# Add a new key
student["school"] = "Metro Middle"
print("After add:   ", student)
Output
After update: {'name': 'Alice', 'age': 15}
After add:    {'name': 'Alice', 'age': 15, 'school': 'Metro Middle'}

Update Multiple Keys with .update()

Python
student = {"name": "Alice", "age": 14}
student.update({"age": 15, "grade": "9th", "gpa": 3.8})
print(student)
Output
{'name': 'Alice', 'age': 15, 'grade': '9th', 'gpa': 3.8}

🗑️ Deleting Items

del — Remove by Key

Python
student = {"name": "Alice", "age": 14, "grade": "8th"}
del student["age"]
print("After del:", student)
Output
After del: {'name': 'Alice', 'grade': '8th'}

pop() — Remove & Return the Value

Python
student = {"name": "Alice", "age": 14, "grade": "8th"}
removed_value = student.pop("grade")
print("Removed:", removed_value)
print("Remaining:", student)
Output
Removed: 8th
Remaining: {'name': 'Alice', 'age': 14}

popitem() — Remove the Last Pair

Python
student = {"name": "Alice", "age": 14, "grade": "8th"}
last = student.popitem()
print("Removed:", last)
print("Remaining:", student)
Output
Removed: ('grade', '8th')
Remaining: {'name': 'Alice', 'age': 14}

clear() — Remove Everything

Python
student = {"name": "Alice", "age": 14, "grade": "8th"}
student.clear()
print("After clear:", student)
Output
After clear: {}

🔄 Iterating Through a Dictionary

There are several ways to loop through a dictionary:

Loop Through Keys (default)

Python
capitals = {"USA": "Washington D.C.", "France": "Paris", "Japan": "Tokyo"}

for country in capitals:
    print(country)
Output
USA
France
Japan

.keys(), .values(), and .items()

Python
capitals = {"USA": "Washington D.C.", "France": "Paris", "Japan": "Tokyo"}

# .keys() — all the keys
print("Keys:  ", list(capitals.keys()))

# .values() — all the values
print("Values:", list(capitals.values()))

# .items() — all key-value pairs as tuples
print("Items: ", list(capitals.items()))
Output
Keys:   ['USA', 'France', 'Japan']
Values: ['Washington D.C.', 'Paris', 'Tokyo']
Items:  [('USA', 'Washington D.C.'), ('France', 'Paris'), ('Japan', 'Tokyo')]

Looping with .items() — Most Useful!

Python
capitals = {"USA": "Washington D.C.", "France": "Paris", "Japan": "Tokyo"}

for country, city in capitals.items():
    print(f"The capital of {country} is {city}")
Output
The capital of USA is Washington D.C.
The capital of France is Paris
The capital of Japan is Tokyo

🔎 Membership & Length

Checking If a Key Exists — in

Python
student = {"name": "Alice", "age": 14, "grade": "8th"}

print("name" in student)
print("email" in student)
print("email" not in student)
Output
True
False
True
⚠️ Important: The in keyword checks keys, not values! To check values, use "Alice" in student.values().

Getting the Length

Python
menu = {"burger": 5.99, "fries": 2.99, "soda": 1.49}
print("Items on menu:", len(menu))
Output
Items on menu: 3

🏗️ Nested Dictionaries

A dictionary can contain other dictionaries as values — great for organizing complex data!

Python
classroom = {
    "student1": {"name": "Alice", "grade": 95},
    "student2": {"name": "Bob", "grade": 87},
    "student3": {"name": "Charlie", "grade": 92}
}

# Access nested values
print(classroom["student1"]["name"])
print(classroom["student2"]["grade"])

# Loop through nested dict
for key, info in classroom.items():
    print(f"{info['name']} scored {info['grade']}")
Output
Alice
87
Alice scored 95
Bob scored 87
Charlie scored 92

🧰 Dictionary Methods — Quick Reference

.keys()Returns all keys
.values()Returns all values
.items()Returns all key-value pairs as tuples
.update(dict2)Adds/updates multiple pairs from another dict
.pop(key)Removes key and returns its value

.clear()Removes all items from the dictionary

📐 Built-in Functions on Dictionaries

Python
scores = {"math": 92, "science": 88, "english": 95, "history": 79}

print("Length:", len(scores))
print("Keys:  ", list(scores.keys()))
print("Max key:  ", max(scores))        # max of KEYS
print("Max value:", max(scores.values()))
print("Min value:", min(scores.values()))
print("Sum:      ", sum(scores.values()))
print("Sorted keys:", sorted(scores))
Output
Length: 4
Keys:   ['math', 'science', 'english', 'history']
Max key:   science
Max value: 95
Min value: 79
Sum:       354
Sorted keys: ['english', 'history', 'math', 'science']
Notice: max(), min(), and sorted() operate on keys by default. Use .values() to work with the values instead!
