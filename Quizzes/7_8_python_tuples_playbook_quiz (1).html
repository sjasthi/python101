What Is a Tuple?

A tuple is a collection of values — just like a list — but with one crucial difference: it is immutable, meaning its contents cannot be changed after it is created.

📅 Real-World Analogy

Think of the days of the week or the elements on the Periodic Table. These things are fixed — they don't change day to day. You wouldn't add an 8th day to the week! A tuple is perfect for storing data like this: ordered, accessible, but permanently locked.

my_tuple = (value1, value2, value3)
Variable name
Values separated by commas
( ) Round brackets — not square!
Python 
days     = ("Monday", "Tuesday", "Wednesday", "Thursday", "Friday")
scores   = (95, 87, 92, 88)
location = (43.65, -79.38)  # latitude, longitude — never changes!

CRUD Operations — What Can You Do?

CRUD stands for Create, Read, Update, Delete. Tuples only support the first two!

C

Create ✅

Create a tuple with initial values

R

Read ✅

Query, traverse, find, search

U

Update ❌

Cannot modify, change, or edit items

D

Delete ❌

Cannot remove individual items (but can delete the whole tuple)

🔒
Immutability — You Can't Change Values

If you try to assign a new value to an index or append to a tuple, Python throws an error.

Python — Trying to modify a tuple 
students = ("abe", "barb", "chris", "dan", "ellie")

# ❌ Try to change index 3 from "dan" to "anna"
students[3] = "anna"

# ❌ Try to append
students.append("frank")
Output
TypeError: 'tuple' object does not support item assignment AttributeError: 'tuple' object has no attribute 'append'
📋
Tuple Methods — Only Two Are Valid

Because tuples are immutable, most list methods simply don't exist for tuples. Only two remain:

MethodPurposeStatus
index(x) Returns the index of the first occurrence of x ✓ Valid
count(x) Counts how many times x appears in the tuple ✓ Valid
append(x)Add x to the end✗ Invalid
extend()Add all items from another list✗ Invalid
insert(i,x)Insert at position✗ Invalid
remove(x)Remove first item x✗ Invalid
pop()Remove last item✗ Invalid
sort()Sort elements✗ Invalid
reverse()Reverse the sequence✗ Invalid
⚠️
Common Trap — Single-Element Tuples

Students often think (x) is a tuple. It's not! Python treats it as regular parentheses.

Python 
x = (5)
print(type(x))   # <class 'int'>   ← NOT a tuple!

y = ("hello")
print(type(y))   # <class 'str'>   ← NOT a tuple!

# ✅ The fix: add a trailing COMMA
a = (5,)
print(type(a))   # <class 'tuple'>  ← THIS is a tuple!
print(a)          # (5,)
⚠️ Remember: It's the comma that makes a tuple — not the brackets. (5,) is a tuple. (5) is just the integer 5.
🔄
Recreating a Tuple — It's Not the Same as Modifying!

You cannot change a tuple — but you can replace it entirely by assigning a brand new tuple to the same variable.

Python 
# Reassign to a brand new tuple
x = (3, 4)
print(x)          # (3, 4)

x = (3, 4, 5)    # x now points to a completely new tuple
print(x)          # (3, 4, 5)

# Concatenate a tuple with itself using +
x = (3, 4)
x = x + x
print(x)          # (3, 4, 3, 4)
Output
💡 Why is this okay? Python creates a brand new tuple in memory each time. The original tuple is never touched — which is why tuples stay truly immutable!
Goal: Access elements in a tuple using indexing and slicing — exactly the same as lists!
Python 
students = ("abe", "barb", "chris", "dan", "ellie")
#  index:      0        1         2          3        4

print(students[0])    # abe        — first element
print(students[4])    # ellie      — last element (index 4)
print(students[-1])   # ellie      — also last element (negative index)
print(students[1:3])  # ('barb', 'chris')  — slice from index 1 up to (not including) 3
print(students[2:])   # ('chris', 'dan', 'ellie')  — from index 2 to the end
Output
💡 Key point: Indexing and slicing work identically to lists. Positive indices count from the left (starting at 0), negative indices count from the right (starting at -1).
Goal: Loop through a tuple and use in / not in membership operators — same as lists!
Python 
planets = ("Mercury", "Venus", "Earth", "Mars")

# Iterating with a for loop
for planet in planets:
    print(planet)

# Membership operators
print("Earth" in planets)       # True
print("Pluto" not in planets)   # True  (Pluto was reclassified!)
print("Mars" not in planets)    # False
Output
Goal: Use Python's built-in functions on tuples — len(), min(), max(), sum(), and sorted().
Python 
scores = (72, 95, 88, 61, 79)

print(len(scores))          # 5   — number of elements
print(min(scores))          # 61  — lowest score
print(max(scores))          # 95  — highest score
print(sum(scores))          # 395 — total
print(sorted(scores))       # [61, 72, 79, 88, 95] — returns a LIST!
print(type(sorted(scores))) # <class 'list'>
Output
⚠️ Note: sorted() returns a list, not a tuple. This makes sense — a sorted result is new data, and sorting would imply changing the tuple!
Goal: Use the only two tuple methods — index(x) and count(x).
Python 
students = ("abe", "barb", "chris", "abe", "dan", "chris", "ellie")
#  index:      0        1         2          3       4         5          6

# index(x) — returns the position of the FIRST match
print(students.index("abe"))    # 0  (first "abe" is at index 0)
print(students.index("chris"))  # 2  (first "chris" is at index 2)

# count(x) — counts how many times x appears
print(students.count("abe"))    # 2  (appears at index 0 and 3)
print(students.count("chris"))  # 2  (appears at index 2 and 5)
print(students.count("dan"))    # 1  (appears only once)
Output
Goal: Understand when to reach for a tuple instead of a list — and see real-world examples.
Python — Good use cases for tuples 
# ✅ Days of the week — never changes
days_of_week = ("Sunday", "Monday", "Tuesday", "Wednesday",
                "Thursday", "Friday", "Saturday")

# ✅ RGB colour — fixed values (Red=255, Green=0, Blue=128)
magenta = (255, 0, 128)

# ✅ GPS coordinates — location doesn't randomly change
toronto = (43.651070, -79.347015)

# ✅ Periodic table element info — always the same
hydrogen = ("H", 1, "Hydrogen")   # symbol, atomic number, name

# Access just like a list
print(f"Today is {days_of_week[1]}")   # Today is Monday
print(f"Red: {magenta[0]}, Green: {magenta[1]}, Blue: {magenta[2]}")
Output
💡 Rule of thumb: Ask yourself — "Should this data ever change?" If the answer is No, use a tuple. If it might change, use a list.
