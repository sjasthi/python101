🔢 Data Types & Operators

See how arithmetic operators work with different data types:

🔢 Integers (int)

a = 15 b = 4 print(a + b) # 19 print(a - b) # 11 print(a * b) # 60 print(a / b) # 3.75 print(a // b) # 3 print(a % b) # 3 print(a ** b) # 50625

📊 Floats (float)

x = 7.5 y = 2.0 print(x + y) # 9.5 print(x - y) # 5.5 print(x * y) # 15.0 print(x / y) # 3.75 print(x // y) # 3.0 print(x % y) # 1.5 print(x ** y) # 56.25

🔀 Mixed Types

int_num = 10 float_num = 3.5 print(int_num + float_num) # 13.5 print(int_num * float_num) # 35.0 print(int_num / float_num) # 2.857... print(int_num // float_num) # 2.0 # Result is always float when mixed

📝 Strings (special cases)

text = "Python" number = 3 # Addition (concatenation) print("Hello " + "World") # Hello World # Multiplication (repetition) print(text * number) # PythonPythonPython print("-" * 10) # ---------- # Other operators don't work with strings!

🎯 Operator Precedence (Order of Operations)

Python follows mathematical order of operations. Higher precedence operators are calculated first:

Precedence Operator Description Example
1 (Highest) ** Exponentiation 2 ** 3 ** 2 = 2 ** (3 ** 2) = 512
2 *, /, //, % Multiplication, Division, Floor Division, Modulo 2 + 3 * 4 = 2 + 12 = 14
3 (Lowest) +, - Addition, Subtraction 5 + 2 - 1 = 7 - 1 = 6

💡 Key Points

🎯 Always Float Division

The / operator always returns a float, even when dividing evenly:

print(6 / 2) # 2.0 (not 2) print(type(6/2)) # <class 'float'>

🔄 Type Mixing

When mixing integers and floats, the result is always a float:

print(5 + 2.0) # 7.0 print(10 * 3.5) # 35.0 print(type(5+2.0)) # <class 'float'>

⚠️ Division by Zero

Dividing by zero causes an error:

print(5 / 0) # ZeroDivisionError print(5 // 0) # ZeroDivisionError print(5 % 0) # ZeroDivisionError

📐 Use Parentheses

Use parentheses to control order of operations:

print(2 + 3 * 4) # 14 print((2 + 3) * 4) # 20 print(2 ** 3 * 4) # 32 print(2 ** (3 * 4)) # 4096
